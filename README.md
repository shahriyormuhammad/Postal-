# Postal

Инструмент для селлеров Wildberries: анализ Индекса Региональной Представленности (ИРП), планирование поставок, распознавание рукописных накладных.

## Возможности

- **Диагностика ИРП** — подключение к WB Statistics API по токену, расчёт покрытия регионов
- **Детали по размерам** — динамика продаж в разрезе артикул × размер × регион
- **Мониторинг прогресса** — снимки ИРП, отслеживание динамики
- **Перераспределение** — рекомендации по перебросу стоков между складами WB
- **Экономика** — оценка экономического эффекта улучшения ИРП
- **План поставки** — создание поставки с ростовкой, экспорт в CSV/XLSX для WB
- **Распознавание накладных** — загрузка фото (в т. ч. рукописных), автоматическое извлечение пар «артикул → коробок» через Tesseract.js

## Запуск

Приложение — один HTML-файл, никакой сборки не требуется.

### Онлайн (GitHub Pages)

Открывается по URL после публикации на GitHub Pages (см. ниже).

### Локально

Откройте `index.html` через локальный HTTP-сервер (не через `file://`, иначе не сработает OCR из-за ограничений Web Workers):

```bash
# Python 3
python -m http.server 8000

# Node.js
npx serve .
```

Затем откройте <http://localhost:8000>.

## Хранение данных

Всё хранится в `localStorage` браузера:

- `irp_state_v1` — настройки кабинетов, токены, данные синхронизации
- `irp_rostovki_v1` — сохранённые ростовки по артикулам
- `irp_nomenclature_v1` — кэш номенклатуры WB (артикулы + баркоды)
- `irp_supply_list_v1` — текущий накопленный список поставки
- `irp_snapshots_v1` — снимки ИРП для мониторинга динамики

Данные **не** отправляются на какой-либо сервер; всё приватно и остаётся в браузере.

## Зависимости

Подгружаются через CDN при использовании:

- [SheetJS](https://sheetjs.com/) (`xlsx@0.18.5`) — экспорт в XLSX
- [Tesseract.js](https://tesseract.projectnaptha.com/) (`5.1.1`) — OCR накладных (lazy-load при первом клике «Распознать»)
- Google Fonts (Manrope, JetBrains Mono)
